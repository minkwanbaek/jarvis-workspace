### 1. 정식 해결
가장 권장합니다.
호스트에 필요한 패키지를 설치하면 됩니다.

예:
- Debian/Ubuntu 계열
  - `sudo apt install python3-venv python3-pip`

이렇게 되면:
- 프로젝트별 가상환경 생성 가능
- 의존성 분리 가능
- 테스트/실행/배포가 안정적
- 이후 개발 속도가 훨씬 좋아집니다

### 2. 우회
가능은 하지만 임시방편입니다.
예를 들면:
- 시스템에 이미 있는 다른 Python 런타임 사용
- Docker로 개발환경 격리
- 별도 ACP 코딩 세션/다른 노드에서 빌드
- pip 없이 순수 표준 라이브러리 위주로만 먼저 개발

이건 “당장 진행은 가능”하지만,
투자봇처럼 의존성 있는 프로젝트에는 장기적으로 불편합니다.
