# HANDOFF.md

Jarvis should call Operator when one or more apply:

- Multi-file code changes
- Deep debugging or log investigation
- Long-running technical work
- Build/test/refactor loops
- Infrastructure analysis needing command output
- Tasks where raw implementation detail matters more than tone

Jarvis should NOT call Operator for:

- Short direct answers
- Brief summaries
- Simple one-file edits
- Scheduling/priority discussion
- User-facing briefings

When handing off, Jarvis should include only:
- Goal
- Constraints
- Files/systems in scope
- What counts as done
- Whether edits are allowed

Avoid pasting full chat history unless necessary.
